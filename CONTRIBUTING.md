# Guía de Contribución

Gracias por tu interés en contribuir a **whisper-local-transcriber**.
Todas las contribuciones son bienvenidas: reportes de errores, propuestas de
mejora, documentación y aportes de código.

Antes de contribuir, por favor revisa el
[Código de Conducta](./CODE_OF_CONDUCT.md).

---

## Formas de contribuir

- Reportar errores
- Proponer nuevas funcionalidades o mejoras
- Mejorar la documentación
- Enviar cambios de código mediante Pull Requests

---

## Reporte de errores

- Revisa los issues existentes antes de crear uno nuevo.
- Usa la plantilla de **Reporte de Error**.
- Incluye, cuando sea posible:
  - Pasos para reproducir el error
  - Comportamiento esperado
  - Comportamiento observado
  - Información del sistema (sistema operativo, versión de Python, etc.)

---

## Propuestas de mejora

- Usa la plantilla de **Solicitud de Funcionalidad**.
- Describe claramente el problema y la solución propuesta.
- Incluye contexto o ejemplos cuando sea relevante.

---

## Commits y Pull Requests

- Escribe mensajes de commit claros y descriptivos.
- Mantén los Pull Requests enfocados y probados.
- Completa la plantilla de Pull Request.
- Mantén una actitud abierta a la retroalimentación.

---

## Calidad del código

- Respeta la estructura y estilo existentes del proyecto.
- Escribe código limpio, legible y mantenible.
- Evita agregar dependencias innecesarias.

---

## Licencia

Al contribuir a este proyecto, aceptas que tus aportes se publiquen bajo la
misma licencia del proyecto.
